How to Use the Analyzer

A complete guide to uploading graphs, configuring the k-WL test, and interpreting your results — including the colour iteration viewer and degree distribution tables.

File Format

Graph files must be plain .txt files. Two formats are accepted:

A
With header (recommended)

First line: n m — total nodes and edges. Then one edge u v per line. Nodes are 0-indexed integers.

6 7
0 1
0 2
1 2
1 3
2 4
3 5
4 5
B
Edge-list only (no header)

Just list edges one per line as u v. Node count is inferred from the maximum index seen.

0 1
0 2
1 2
1 3
2 4

Lines beginning with # or % are treated as comments and ignored. Both space-separated and comma-separated formats are accepted.

Step-by-Step Guide

1
Choose Analysis Mode

Single Graph runs k-WL on one graph and shows the colour refinement. Compare runs k-WL on two graphs simultaneously using a shared hash map — required for a valid isomorphism test.

2
Set k (Refinement Depth)

Use the − and + buttons to choose k from 1 to 5. The safe-limits table below updates to show the maximum graph size for that k. Start with k=1 for large graphs, increase for more distinguishing power.

3
Upload Graph File(s)

Click the upload zone or drag and drop a .txt file. In Compare mode, upload Graph A and Graph B separately. The zone turns green when a file is loaded.

4
Run Analysis

Click Run Analysis. A progress bar tracks parsing, refinement, and visualization. The algorithm enforces the safe-limits table — if your graph is too large for the chosen k, a clear error explains what to do.

5
Read the Results

Results appear in four sections: Analysis Summary (counts), Isomorphism Verdict (compare mode), Color Iterations (interactive graph viewer), and Degree Distribution (per-node tables).

Reading the Results

Isomorphism Verdict — shown only in Compare mode. A green ✓ Isomorphic means k-WL cannot distinguish the graphs (they may be isomorphic). A red ✗ Non-Isomorphic is conclusive: the graphs are definitely different. The three signal cards show:

  • Degree Sequence — do both graphs have the same sorted degree multiset?
  • k-WL Certificate — do the final tuple colour histograms match?
  • Divergence — which iteration first produced different node colour distributions?

Color Iterations viewer — click the iteration tabs (Initial, Iter 1, Iter 2 …) to step through each round of colour refinement. Node colours update to reflect their colour class at that iteration. Hover any node to see its id, colour class, and degree. Drag nodes to rearrange the layout.

Degree Distribution table — lists every node and its degree. Nodes with a unique degree (degree that no other node shares) are highlighted in red — these are structural anomalies that often make two graphs easy to distinguish.

Understanding the Size Limits

k-WL operates on all nk ordered k-tuples. This grows explosively with k:

  • k=1, n=1000 → 1,000 tuples ✓
  • k=2, n=100 → 10,000 tuples ✓
  • k=3, n=50 → 125,000 tuples ✓ (but slow)
  • k=3, n=200 → 8,000,000 tuples ✗ browser will freeze
  • k=5, n=20 → 3,200,000 tuples — already pushing limits

The analyzer blocks analysis before it starts if your graph exceeds the safe limit for the chosen k, and suggests a lower k or a smaller graph. The limits in the sidebar are conservative — they ensure a responsive browser experience on typical hardware.
